# Context for gwt-0005: fork.server.forwards-resume

## Behavior
Given mapCliOptionsToSDK receives an options object where options.resume is a non-empty string (a parent sessionId), when it constructs sdkOptions, then sdkOptions.resume is set to options.resume — the resume field is additively copied without altering any other existing field in sdkOptions.

## Key Invariants to Model
- ResumeForwarded: if input.resume is non-empty, output.resume === input.resume.
- ResumeAbsent: if input.resume is missing/empty, output.resume is undefined (property not set).
- NoClobber: pre-existing sdkOptions fields (cwd, permissionMode, allowedTools, disallowedTools, tools, model) are unchanged by the resume copy.

## Modeling Approach
- CONSTANTS: InputOptions variants (with/without resume).
- Variables: sdkOptions.
- Actions: CallMap(options) → produces sdkOptions.
- Invariants: ResumeForwarded, ResumeAbsent, NoClobber.

## Anti-Patterns (DO NOT USE)
- Booting the full Express/WebSocket server for this unit test — call mapCliOptionsToSDK directly.
- Asserting `sdk.resume === options.sessionId` — that's a different field (parent's id is in `resume`, not `sessionId`).
- Setting `sdkOptions.resume` unconditionally (to '' or undefined) — invariant ResumeAbsent requires the property not to be set when input.resume is falsy.
