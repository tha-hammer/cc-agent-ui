# Context for gwt-0003: fork.handler.sends-claude-command-with-fork-options

## Behavior
Given handleForkFromMessage has generated a new sessionId and resolved all inherited settings from the current composer state, when it dispatches the fork request over WebSocket, then it calls sendMessage with a payload of shape { type: 'claude-command', command: '', options: { ...inheritedSettings, sessionId: newUUID, resume: parentSessionId, forkSession: true, resumeSessionAt: messageId } } — using the existing 'claude-command' channel with no new WS message type.

## Concrete Data Shapes

```ts
interface ClaudeForkPayload {
  type: 'claude-command';
  command: '';                   // empty string — no new prompt
  options: {
    // inherited from composer state:
    provider?: 'claude';
    permissionMode?: 'default' | 'plan' | 'acceptEdits' | 'bypassPermissions';
    projectPath?: string;
    cwd?: string;
    toolsSettings?: { allowedTools: string[]; disallowedTools: string[]; skipPermissions: boolean };
    // fork-specific:
    sessionId: string;           // crypto.randomUUID()
    resume: string;              // parentSessionId
    forkSession: true;
    resumeSessionAt: string;     // messageId === SDKAssistantMessage.uuid

}
```

## Key Invariants to Model
- ChannelReuse: payload.type === 'claude-command' (no new WS message type).
- ForkMarker: options.forkSession === true AND options.resume === parentSessionId AND options.resumeSessionAt === messageId.
- FreshSessionId: options.sessionId is a new UUID distinct from parentSessionId.
- EmptyCommand: payload.command === '' (fork does not inject a new prompt).

## Modeling Approach
- CONSTANTS: ParentId, MessageId, NewUUID.
- Variables: lastSentPayload.
- Actions: InvokeFork → SendMessage(payload).
- Invariants: ChannelReuse, ForkMarker, FreshSessionId, EmptyCommand.

```tsx
describe('gwt-0003 fork.handler.sends-claude-command-with-fork-options', () => {
  it('dispatches claude-command with fork options', () => {
    const parentId = 'parent-xyz';
    const messageId = 'a-uuid-1234';
```

## Anti-Patterns (DO NOT USE)
- Inventing a new WS type like `'fork-command'` — the GWT explicitly reuses `claude-command`.
- Asserting `sessionId: parentId` — that's the parent, not the new fork.
- Forgetting `forkSession: true` — server gate at claude-sdk.js:146 will drop sessionId without it.
- Sending a non-empty `command` string — fork must not inject a new user prompt.
- Using `useSessionStore.getState()` or `useSessionStore.getState().sessions.set(...)` — `useSessionStore` is NOT a Zustand store. Seed via `appendRealtime` inside `renderHook` + `act`.
