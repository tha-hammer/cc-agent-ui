# Context for gwt-0001: fork.pane.passes-index-and-onfork

## Behavior
Given ChatMessagesPane is mapping visibleMessages to MessageComponent elements and an onFork prop has been threaded from the parent, when ChatMessagesPane renders each MessageComponent at array index i, then each MessageComponent receives messageIndex=i and onFork=the threaded onFork callback, in addition to all previously-passed props.

## Concrete Data Shapes
```ts

  createDiff: (...) => unknown;

  // NEW (added by this feature):
  onFork: (messageId: string, messageIndex: number) => void;
}

// Expected (post-change) MessageComponent props must include:
// messageIndex: number
// onFork: (messageId: string, messageIndex: number) => void
```

## Key Invariants to Model
- IndexFidelity: the index forwarded to MessageComponent equals its array position in visibleMessages.
- CallbackIdentity: the onFork callback reference passed down is the exact reference received from parent (no wrapping).
- PropAdditivity: previously-passed props (message, prevMessage, createDiff, onFileOpen, etc.) remain unchanged.
- NoSideEffects: rendering does not mutate visibleMessages.

## Modeling Approach
- CONSTANTS: MessageCount (length of visibleMessages), OnForkRef (symbolic fn ref).
- Variables: renderedChildren (sequence of prop bundles passed to MessageComponent).
- Actions: RenderPane iterates 0..MessageCount-1 producing a bundle per index.
- Invariants: IndexFidelity, CallbackIdentity, PropAdditivity across all indices.

## Test Interface (MANDATORY — concrete code)
```tsx
// tests/generated/gwt-0001.test.tsx
import { describe, it, expect, vi } from 'vitest';
import { render } from '@testing-library/react';
import ChatMessagesPane from '../../src/components/chat/view/subcomponents/ChatMessagesPane';

vi.mock('../../src/components/chat/view/subcomponents/MessageComponent', () => ({
  default: vi.fn(() => null),
}));
import MessageComponent from '../../src/components/chat/view/subcomponents/MessageComponent';

describe('gwt-0001 fork.pane.passes-index-and-onfork', () => {
  it('passes messageIndex=i and onFork to each MessageComponent', () => {
    const onFork = vi.fn();
    const visibleMessages = [
      { id: 'm0', type: 'user', content: 'hi', timestamp: Date.now() },
      { id: 'm1', type: 'assistant', content: 'hello', timestamp: Date.now() },
      { id: 'm2', type: 'assistant', content: 'again', timestamp: Date.now() },
    ];

    render(
      <ChatMessagesPane
        visibleMessages={visibleMessages as any}
        createDiff={() => null}
        onFileOpen={() => {}}
        onShowSettings={() => {}}
        onGrantToolPermission={() => {}}
        autoExpandTools={false}
        showRawParameters={false}
        showThinking={false}
        selectedProject={{ fullPath: '/p', path: '/p' } as any}
        provider="claude"
        loadEarlierMessages={() => {}}
        loadAllMessages={() => {}}
        isLoading={false}
        onFork={onFork}
      />
    );

    const calls = (MessageComponent as any).mock.calls;
    expect(calls.length).toBe(3);
    calls.forEach(([props]: any[], i: number) => {
      expect(props.messageIndex).toBe(i);
      expect(props.onFork).toBe(onFork);
      expect(props.message).toBe(visibleMessages[i]);
    });
  });
});
```

## Anti-Patterns (DO NOT USE)
- Asserting on DOM text for the fork button — this GWT is about prop wiring, not rendering.
- Wrapping onFork in an inline arrow (`onFork={(id, i) => onFork(id, i)}`) in the component; that breaks CallbackIdentity.
- Using `message.uuid` — the normalized message shape uses `message.id`.
