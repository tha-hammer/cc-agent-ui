# Context for gwt-0011: fork.parent.session-unchanged-after-fork

## Behavior
Given a fork is initiated from message index N of parent session P, when the fork command is dispatched and the new session is navigated to, then `useSessionStore.sessions.get(P).merged` still contains all of its original messages unmodified — no append, splice, or clear operation is issued against the parent session slot.

## Concrete Data Shapes

```ts
// Fork handler payload

```

## Key Invariants to Model
- ParentMergedFrozen: ∀ step after `handleForkFromMessage(P, mid, N)`: `store.sessions.get(P).merged === preFork.merged` (same array reference or deep-equal — no mutation).
- NoParentWrites: no `appendRealtime(P, …)` / `updateStreaming(P, …)` / `finalizeStreaming(P)` call during the fork dispatch path.
- NewSessionIsolated: all realtime writes after `session_created` target `newId ≠ P`.

## Modeling Approach
- CONSTANTS: P (parent id), NEW_ID, N (fork index).
- Variables: `store` (map sessionId → merged[]), `writesLog` (sequence of (sid, op)).
- Actions: `Fork(P,N)` (emits WS command, receives session_created + replays under NEW_ID).
- Invariants: ParentMergedFrozen, NoParentWrites.

## Test Interface (MANDATORY — concrete code)
Integration spec with a stubbed SDK. Asserts the in-memory session store, not disk — the server does NOT write the parent's JSONL during fork (SDK fork reads parent JSONL at `~/.claude/projects/{proj}/{P}.jsonl` read-only, and writes a new JSONL under the forked id).

```tsx
// cc-agent-ui/tests/generated/gwt-0011.test.tsx
import { describe, it, expect, vi } from 'vitest';
import { renderHook, act } from '@testing-library/react';
import { useSessionStore } from '../../src/stores/useSessionStore';

// NOTE: useSessionStore is a plain React hook (NOT a Zustand store).
// It has no .getState() / no exposed .sessions Map. Seed and read via the real
// hook API (appendRealtime / getMessages) called inside renderHook + act.
//
// SDK stub: if this test drives the fork pipeline end-to-end via queryClaudeSDK,
// reuse the `vi.mock('@anthropic-ai/claude-agent-sdk', ...)` stub from
// gwt-0013's Test Interface section (see gwt-0013.md). For this in-memory
// parent-isolation test we only need direct hook calls.

const P   = '11111111-1111-4111-8111-111111111111';
const NEW = '22222222-2222-4222-8222-222222222222';
const MID = 'asst-uuid-aaaa';

describe('gwt-0011 fork.parent.session-unchanged-after-fork', () => {
  it('parent merged array is not mutated when fork dispatched + session_created received', async () => {
    // Seed parent store via the real hook API.
    const { result: storeResult } = renderHook(() => useSessionStore());
    const parentSeed = [
      { id: 'u1', kind: 'text', role: 'user',      content: 'hi',   provider: 'claude', sessionId: P, timestamp: '' },
      { id: 'a1', kind: 'text', role: 'assistant', content: 'hey',  provider: 'claude', sessionId: P, timestamp: '' },
      { id: MID, kind: 'text', role: 'assistant', content: 'more', provider: 'claude', sessionId: P, timestamp: '' },
    ];
    act(() => {
      for (const m of parentSeed) storeResult.current.appendRealtime(P, m as any);
    });

    const parentSnapshot = [...storeResult.current.getMessages(P)];
    const appendSpy = vi.spyOn(storeResult.current, 'appendRealtime');

    // Simulate fork replay: SDK events arrive scoped to NEW session id.
    const replay = [
      { id: 'r1', kind: 'text' as const, role: 'user',      content: 'hi',  provider: 'claude' as const, sessionId: NEW, timestamp: '' },
      { id: 'r2', kind: 'text' as const, role: 'assistant', content: 'hey', provider: 'claude' as const, sessionId: NEW, timestamp: '' },
    ];
    act(() => {
      for (const m of replay) storeResult.current.appendRealtime(NEW, m as any);
    });

    // Parent unchanged — read via hook API.
    const after = storeResult.current.getMessages(P);
    expect(after).toEqual(parentSnapshot);
    expect(after.length).toBe(3);

    // No parent writes
    for (const call of appendSpy.mock.calls) {
      expect(call[0]).not.toBe(P);
    }
  });
});
```

## Anti-Patterns (DO NOT USE)
- Calling `store.sessions.get(P).merged.splice(index)` to "trim" the parent — the parent must remain full.
- `store.clearSession(P)` on fork — parent is untouched.
- Routing SDK replay events back to P instead of the new sessionId — realtime handler must resolve `sid` from `msg.sessionId`, not from the outgoing command's `resume`.
- Using `useSessionStore.getState()` / `useSessionStore.getState().sessions.set(...)` / `useSessionStore.getState().sessions.get(...)` — `useSessionStore` is NOT a Zustand store. Use `renderHook(() => useSessionStore())` + `appendRealtime` / `getMessages` via `storeResult.current`.
- Calling `store.createSession(...)` — no such method exists. Session slots are lazy-created on first `appendRealtime` / `getSlot` call.
