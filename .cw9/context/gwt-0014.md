# Context for gwt-0014: fork.spike.error-path-on-bad-uuid

## Behavior
Given `mapCliOptionsToSDK` forwards a `resumeSessionAt` value that the Claude SDK rejects (e.g., not a known message UUID, or a synthetic `claude_`-prefixed id that slipped through), when the SDK throws inside the async generator at `claude-sdk.js`, then the existing catch block at `claude-sdk.js:694-716` emits a `{ kind: 'error', content: <string>, provider: 'claude', sessionId }` event over WebSocket — no new pre-flight validation is added; the existing error path covers this case.

## Concrete Data Shapes
```ts
// Fork options with bad uuid
type BadForkOptions = {
  resume: string;               // valid parent UUID
  forkSession: true;
  resumeSessionAt: 'claude_bad_prefix' | 'not-a-uuid';   // SDK will reject
  sessionId: string;            // new UUID
  cwd: string;
  toolsSettings: { allowedTools: []; disallowedTools: []; skipPermissions: false };
};

// Expected WS error message (createNormalizedMessage contract, types.js:111)
type ErrorMsg = {
  id: string;                   // auto-generated: `error_<uuid>`
  timestamp: string;
  provider: 'claude';
  sessionId: string | null;     // parent id if captured, else null
  kind: 'error';
  content: string;              // error.message
};
```

## Key Invariants to Model
- ErrorEmittedOnThrow: if SDK generator throws at iteration, exactly one `{ kind: 'error', provider: 'claude' }` message is sent on `ws`.
- NoHalfState: after error, no `{ kind: 'complete' }` is sent for the same session.
- ErrorContentIsMessage: `errorMsg.content === thrownError.message` (exactly — see claude-sdk.js:706).
- NoPreflightNeeded: `mapCliOptionsToSDK` does NOT validate UUID format; it is a pure passthrough for `resumeSessionAt`.

## Modeling Approach
- CONSTANTS: BAD_UUID, PARENT_ID, THROWN_MSG.
- Variables: `wsOut` (sequence of messages), `errorCount`, `completeCount`.
- Actions: `StartQuery`, `SDKYieldThrow(err)`, `CatchEmit`.
- Invariants: ErrorEmittedOnThrow, NoHalfState, ErrorContentIsMessage.

## Test Interface (MANDATORY — concrete code)
```ts
// cc-agent-ui/tests/generated/gwt-0014.test.ts
import { describe, it, expect, vi } from 'vitest';

const PARENT  = '11111111-1111-4111-8111-111111111111';
const NEW     = '22222222-2222-4222-8222-222222222222';
const BAD_AT  = 'claude_not_a_uuid_at_all';
const THROWN  = 'Invalid resumeSessionAt: message UUID not found';

vi.mock('@anthropic-ai/claude-agent-sdk', () => {
  async function* throwingQuery() {
    // SDK rejects bad resumeSessionAt on first pull
    throw new Error(THROWN);
    // eslint-disable-next-line no-unreachable
    yield undefined as any;
  }
  return { query: vi.fn(() => throwingQuery()) };
});

import { queryClaudeSDK } from '../../server/claude-sdk.js';

describe('gwt-0014 fork.spike.error-path-on-bad-uuid', () => {
  it('emits {kind:error, content, provider:claude} when SDK throws on bad resumeSessionAt', async () => {
    const sent: any[] = [];
    const ws = {
      send: (m: any) => sent.push(m),
      setSessionId: vi.fn(),
      userId: null,
    };

    await expect(queryClaudeSDK('', {
      sessionId: NEW,
      resume: PARENT,
      forkSession: true,
      resumeSessionAt: BAD_AT,
      cwd: '/tmp/proj',
      toolsSettings: { allowedTools: [], disallowedTools: [], skipPermissions: false },
    }, ws as any)).rejects.toThrow(THROWN);  // queryClaudeSDK re-throws after emit (claude-sdk.js:715)

    const errors = sent.filter((m) => m.kind === 'error');
    expect(errors).toHaveLength(1);
    expect(errors[0]).toMatchObject({
      kind: 'error',
      content: THROWN,
      provider: 'claude',
    });

    // No complete for the failed run
    expect(sent.some((m) => m.kind === 'complete')).toBe(false);
  });

  it('mapCliOptionsToSDK does NOT pre-validate resumeSessionAt', async () => {
    const { mapCliOptionsToSDK } = await import('../../server/claude-sdk.js');
    expect(() =>
      mapCliOptionsToSDK({
        sessionId: NEW, resume: PARENT, forkSession: true, resumeSessionAt: BAD_AT,
      }),
    ).not.toThrow();
    const out = mapCliOptionsToSDK({
      sessionId: NEW, resume: PARENT, forkSession: true, resumeSessionAt: BAD_AT,
    });
    expect(out.resumeSessionAt).toBe(BAD_AT);     // passthrough confirmed
  });
});
```

## Callers
- Catch block at `claude-sdk.js:694`: protects the entire `for await (const message of queryInstance)` loop; covers both map/query construction errors and mid-stream throws.
- `ws.send(createNormalizedMessage({ kind: 'error', content: error.message, ... }))` at `claude-sdk.js:706`.
  Reuse risk: low — the catch is the canonical error path for all SDK failures, not just fork.

## Anti-Patterns (DO NOT USE)
- Adding a regex-based UUID validator to `mapCliOptionsToSDK` — duplicative; SDK validates and the catch is sufficient (per GWT contract).
- Swallowing the rethrow at line 715 — tests and upstream handlers rely on the promise rejection in addition to the WS error message.
- Emitting `{ kind: 'complete', exitCode: 1 }` in the catch — current code only sends `kind: 'error'`; adding `complete` would change the frontend state machine (see `useChatRealtimeHandlers.ts:261 complete` branch vs `:305 error` branch).
- Setting `errorMsg.content` to a custom string (e.g. "Fork failed") instead of `error.message` — the GWT requires `content: <string>` derived from the thrown error, matching the existing line-706 behavior.
