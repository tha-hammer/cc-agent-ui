# Context for gwt-0004: fork.handler.inherits-settings

## Behavior
Given handleForkFromMessage is invoked for a Claude session, when it builds the inheritedSettings portion of the payload, then it reads provider, permissionMode, projectPath, and cwd from the current composer state, and calls getToolsSettings('claude') to retrieve toolsSettings (allowedTools, disallowedTools, skipPermissions) from localStorage — no settings prompt or dialog is shown.

## Concrete Data Shapes
```ts
// Source reference: useChatComposerState.ts:579-602 (getToolsSettings pattern)
interface InheritedSettings {
  provider: 'claude';
  permissionMode: 'default' | 'plan' | 'acceptEdits' | 'bypassPermissions';
  projectPath: string;            // selectedProject.fullPath || selectedProject.path
  cwd: string;                    // same as projectPath in practice
  toolsSettings: {
    allowedTools: string[];
    disallowedTools: string[];
    skipPermissions: boolean;
  };
}

// localStorage key for claude: 'claude-settings'
// Default if missing: { allowedTools: [], disallowedTools: [], skipPermissions: false }
```

## Key Invariants to Model
- NoDialog: no settings modal / dialog component is rendered or opened during fork.
- LocalStorageRead: `localStorage.getItem('claude-settings')` is the sole source for toolsSettings.
- ComposerStatePull: provider/permissionMode/projectPath/cwd come from composer hook state, not prompts.
- ProviderKey: the storage key selected is exactly `'claude-settings'` (not cursor/codex/gemini variants).

## Modeling Approach
- CONSTANTS: StoredClaudeSettings, ComposerState.
- Variables: payloadOptions.toolsSettings, payloadOptions.provider, dialogsOpen.
- Actions: InvokeFork reads storage + composer state; no dialog action fires.
- Invariants: NoDialog, LocalStorageRead, ComposerStatePull, ProviderKey.

// NOTE: useSessionStore is a plain React hook (NOT a Zustand store). Seed via

describe('gwt-0004 fork.handler.inherits-settings', () => {
  beforeEach(() => {
    localStorage.clear();
    localStorage.setItem('claude-settings', JSON.stringify({
      allowedTools: ['Read', 'Grep'],

      skipPermissions: false,

      } as any);
    });

    const sendMessage = vi.fn();

    // Project shape: real Project type requires `name`, `displayName`, `fullPath`
    // (path optional). See src/types/app.ts:33.
    const { result } = renderHook(() =>

    expect(opts.projectPath).toBe('/work/proj');

## Anti-Patterns (DO NOT USE)
- Mocking `safeLocalStorage` module instead of spying on `Storage.prototype.getItem` — the real code uses a `safeLocalStorage` wrapper that ultimately calls localStorage; assert on the storage key.
- Reading from `cursor-tools-settings` / `codex-settings` / `gemini-settings` — wrong provider branch.
- Opening a settings modal dialog before sending (fork is fire-and-forget from current composer state).
- Hardcoding `toolsSettings: { allowedTools: [], ... }` default when storage has a value.
- Using `useSessionStore.getState()` or `useSessionStore.getState().sessions.set(...)` — `useSessionStore` is NOT a Zustand store. Seed via `appendRealtime` inside `renderHook` + `act`.
