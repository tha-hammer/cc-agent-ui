# Context for gwt-0013: fork.spike.sdk-replay-events-arrive

## Behavior
Given a server-side smoke-test posts a fork command `{ resume: parentSessionId, forkSession: true, resumeSessionAt: assistantMessageUUID, sessionId: newUUID, cwd }` against a real 3-message Claude session, when the async generator from `query()` is consumed, then (a) events stream back without throwing, (b) at least one event carries a new `session_id` distinct from the parent, (c) `claudeAdapter.normalizeMessage` handles each replay event (`SDKUserMessageReplay` with `isReplay: true`) without throwing or producing malformed `NormalizedMessage` objects.

## Concrete Data Shapes
```ts

  id: string; sessionId: string; timestamp: string; provider: 'claude';

```

## Key Invariants to Model
- StreamTerminates: generator consumption completes (reaches `result` event) without thrown error.
- NewSessionIdObserved: ∃ event with `session_id !== parentId`.
- AdapterTotal: for every replay event e, `normalizeMessage(e, sid)` returns `NormalizedMessage[]` (possibly empty) where each has string `id`, string `sessionId`, `provider === 'claude'`.
- ReplayIdentityPreserved: when a replay event carries `uuid`, the resulting NormalizedMessage's `id` starts with that uuid (see adapter.js:34 `baseId = raw.uuid || …`).

## Modeling Approach
- CONSTANTS: PARENT_ID, NEW_ID, ASSISTANT_UUID, REPLAY_EVENTS (finite sequence).
- Variables: `consumedEvents`, `normalizedOut`, `observedSessionIds`.
- Actions: `YieldNext(event)` → pushes through adapter.
- Invariants: StreamTerminates, NewSessionIdObserved, AdapterTotal.

## Test Interface (MANDATORY — concrete code)
Runnable as a vitest integration test. Stubs `query()` with a canned async generator — no real SDK/network required for the spike; exercises the real adapter and the real `queryClaudeSDK` control flow via module mocking.

```ts
// cc-agent-ui/tests/generated/gwt-0013.test.ts
import { describe, it, expect, vi } from 'vitest';

const PARENT = '11111111-1111-4111-8111-111111111111';
const NEW    = '22222222-2222-4222-8222-222222222222';
const ASST   = 'aaaaaaaa-aaaa-4aaa-8aaa-aaaaaaaaaaaa';

// Stub the SDK before importing claude-sdk.js
vi.mock('@anthropic-ai/claude-agent-sdk', () => {
  async function* fakeQuery() {
    // Replay: user msg (was the original turn-1 user input)
    yield { type: 'user', isReplay: true, uuid: 'r-u1',
            message: { role: 'user', content: 'hi' },
            parent_tool_use_id: null, session_id: NEW };
    // Replay: assistant msg
    yield { type: 'assistant', uuid: 'r-a1', parent_tool_use_id: null,
            message: { id: 'm-a1', type: 'message', role: 'assistant',
                       content: [{ type: 'text', text: 'hey' }],
                       model: 'claude-sonnet-4.5', stop_reason: 'end_turn',
                       stop_sequence: null,
                       usage: { input_tokens: 1, output_tokens: 1 } },
            session_id: NEW };
    // Fresh assistant turn (post-replay)
    yield { type: 'assistant', uuid: 'a2', parent_tool_use_id: null,
            message: { id: 'm-a2', type: 'message', role: 'assistant',
                       content: [{ type: 'text', text: 'fresh' }],
                       model: 'claude-sonnet-4.5', stop_reason: 'end_turn',
                       stop_sequence: null,
                       usage: { input_tokens: 1, output_tokens: 1 } },
            session_id: NEW };
    yield { type: 'result', subtype: 'success', session_id: NEW,
            modelUsage: {}, duration_ms: 1, duration_api_ms: 1,
            is_error: false, num_turns: 1, total_cost_usd: 0 };
  }
  return { query: vi.fn(() => fakeQuery()) };
});

import { queryClaudeSDK } from '../../server/claude-sdk.js';
import * as claudeAdapter from '../../server/providers/claude/adapter.js';

describe('gwt-0013 fork.spike.sdk-replay-events-arrive', () => {
  it('consumes the fork stream, normalizes replay events, observes new session_id', async () => {
    const sent: any[] = [];
    const ws = {
      send: (m: any) => sent.push(m),
      setSessionId: vi.fn(),
      userId: null,
    };

    await expect(queryClaudeSDK('', {
      sessionId: NEW,
      resume: PARENT,
      forkSession: true,
      resumeSessionAt: ASST,
      cwd: '/tmp/proj',
      toolsSettings: { allowedTools: [], disallowedTools: [], skipPermissions: false },
    }, ws as any)).resolves.not.toThrow();

    // (a) stream terminated — got a complete event
    expect(sent.some(m => m.kind === 'complete')).toBe(true);

    // (b) at least one event carried a session_id distinct from parent
    expect(sent.some(m => m.sessionId === NEW)).toBe(true);
    expect(sent.every(m => m.sessionId !== PARENT || m.sessionId === null)).toBe(true);

    // (c) adapter produces well-formed NormalizedMessages for replay events
    const replayEvent = {
      type: 'user', isReplay: true, uuid: 'r-u1',
      message: { role: 'user', content: 'hi' },
      parent_tool_use_id: null, session_id: NEW,
    };
    const out = claudeAdapter.normalizeMessage(replayEvent, NEW);
    expect(Array.isArray(out)).toBe(true);
    for (const nm of out) {
      expect(typeof nm.id).toBe('string');
      expect(nm.id.length).toBeGreaterThan(0);
      expect(nm.sessionId).toBe(NEW);
      expect(nm.provider).toBe('claude');
    }
  });
});
```

## Anti-Patterns (DO NOT USE)
- Hitting the real Claude API in CI — use the `vi.mock` generator above.
- Asserting on streaming-delta ordering — the spike only needs existence of one new-session event + no throws.
- Treating `SDKUserMessageReplay` as a distinct `kind` in the adapter — it flows through the existing `raw.message?.role === 'user'` branch (adapter.js:37); the `isReplay` flag is passthrough, not a new code path.
- Forgetting to pass `sessionId` to the stubbed `ws.send` normalized messages — downstream store routing keys on `msg.sessionId`.
- Using `useSessionStore.getState()` anywhere else in the fork suite — `useSessionStore` is NOT a Zustand store. This GWT doesn't touch the store, but sibling GWTs (0002/0003/0004/0011) that do must seed via `renderHook(() => useSessionStore())` + `appendRealtime` inside `act`.
