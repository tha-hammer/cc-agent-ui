# Context for gwt-0007: fork.server.forwards-resume-session-at

## Behavior
Given mapCliOptionsToSDK receives an options object where options.resumeSessionAt is a non-empty UUID string (the SDK assistant message uuid), when it constructs sdkOptions, then sdkOptions.resumeSessionAt is set to options.resumeSessionAt, matching the SDK's resumeSessionAt?: string contract at sdk.d.ts:840.

## Key Invariants to Model
- ResumeAtForwarded: non-empty input.resumeSessionAt ⇒ output.resumeSessionAt equals input value byte-for-byte.
- ResumeAtAbsent: missing/empty input.resumeSessionAt ⇒ output.resumeSessionAt not set.
- StringIdentity: the copy is reference-equal (no trimming, lowercasing, reformatting).
- NoClobber: other sdkOptions fields unchanged by this copy.

## Anti-Patterns (DO NOT USE)
- Validating UUID format inside mapCliOptionsToSDK — the GWT and SDK contract do not require pre-flight validation; error path at claude-sdk.js:694-706 catches SDK rejection.
- Lowercasing or trimming the UUID — violates StringIdentity.
- Forwarding `resumeSessionAt` when it's an empty string — violates ResumeAtAbsent (use truthy guard `if (options.resumeSessionAt) ...`).
- Asserting on a synthetic `claude_xxx` id — client gate blocks those; server mapping is oblivious.
