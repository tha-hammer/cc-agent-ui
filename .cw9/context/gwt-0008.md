# Context for gwt-0008: fork.server.forwards-session-id-only-when-forking

## Behavior
Given `mapCliOptionsToSDK` receives an options object where `options.sessionId` is set to a UUID string, when it constructs `sdkOptions`, then `sdkOptions.sessionId` is set to `options.sessionId` IFF `options.forkSession` is also truthy; when `forkSession` is absent or false, the caller-supplied `sessionId` is NOT forwarded into `sdkOptions` (matches SDK constraint at sdk.d.ts:834: "Cannot be used with `continue` or `resume` unless `forkSession` is also set").

## Key Invariants to Model
- SessionIdGatedByFork: `sdkOptions.sessionId` is defined ⇒ `input.forkSession === true && typeof input.sessionId === 'string'`.
- AdditiveMapping: setting fork fields does not mutate or delete existing `cwd`/`permissionMode`/`allowedTools`/`disallowedTools`/`model`/`systemPrompt`.
- NoLeakWithoutFork: `input.sessionId` set AND `input.forkSession !== true` ⇒ `sdkOptions.sessionId === undefined`.

## Modeling Approach
- CONSTANTS: UUID_A, UUID_B (distinct non-empty strings).
- Variables: `input` (record with optional sessionId, forkSession, resume), `output` (sdkOptions record).
- Actions: `Map(input)` → compute output per mapping function.
- Invariants: SessionIdGatedByFork, AdditiveMapping, NoLeakWithoutFork.

## Test Interface (MANDATORY — concrete code)
Requires `mapCliOptionsToSDK` to be exported from `server/claude-sdk.js` (currently module-internal — add `export { mapCliOptionsToSDK }`).

```ts
// cc-agent-ui/tests/generated/gwt-0008.test.ts
import { describe, it, expect } from 'vitest';
import { mapCliOptionsToSDK } from '../../server/claude-sdk.js';

const PARENT = '11111111-1111-4111-8111-111111111111';
const FORKED = '22222222-2222-4222-8222-222222222222';

describe('gwt-0008 fork.server.forwards-session-id-only-when-forking', () => {
  it('sets sdkOptions.sessionId when forkSession=true', () => {
    const out = mapCliOptionsToSDK({
      sessionId: FORKED,
      resume: PARENT,
      forkSession: true,
      resumeSessionAt: 'asst-uuid',
      cwd: '/tmp/proj',
    });
    expect(out.sessionId).toBe(FORKED);
    expect(out.forkSession).toBe(true);
    expect(out.resume).toBe(PARENT);
  });

  it('does NOT forward sessionId when forkSession is absent', () => {
    const out = mapCliOptionsToSDK({ sessionId: FORKED, cwd: '/tmp/proj' });
    expect(out.sessionId).toBeUndefined();
  });

  it('does NOT forward sessionId when forkSession=false', () => {
    const out = mapCliOptionsToSDK({ sessionId: FORKED, forkSession: false });
    expect(out.sessionId).toBeUndefined();
  });

  it('current behavior: resume-only still loads history (legacy path)', () => {
    // Pre-fork behavior: `sessionId` alone was historically mapped to sdkOptions.resume.
    // New mapping MUST NOT break: passing only sessionId (no forkSession) still yields resume.
    const out = mapCliOptionsToSDK({ sessionId: PARENT });
    expect(out.resume).toBe(PARENT);
    expect(out.sessionId).toBeUndefined();
  });
});
```

## Anti-Patterns (DO NOT USE)
- Forwarding `sessionId` unconditionally (violates SDK contract — SDK throws when `sessionId` is supplied without `forkSession` alongside `resume`).
- Overwriting `sdkOptions.resume` with `sessionId` — breaks legacy `{ sessionId }`-only resume behavior (see last test).
- Using `options.sessionId && options.forkSession` but forgetting to also set `sdkOptions.forkSession = true`.
